非同期学習の今後の進め方 (実アプリ中心)

1. 通知配信ミニアプリを作る
- 目的: Queue と worker の基本を実装して定着させる
- 必須要素: Queue / worker / None終了シグナル / queue.join / gather
- 完了条件: 投稿を受けて、複数workerで配信し、最終集計を出せる

2. 画像アップロードAPIを作る
- 目的: 実務でよく使う制御を追加する
- 必須要素: Semaphore / timeout / retry / to_thread / Lock
- 完了条件: 成功・入力不正・失敗を分けて集計できる

3. リアルタイム更新ダッシュボードを作る
- 目的: ストリーム処理の感覚を身につける
- 必須要素: async for / 定期受信 or イベント受信 / キャンセル設計
- 完了条件: 受信しながら画面(またはログ)を更新できる

実装の順番(毎回同じでOK)
1) 最小機能だけ作る
2) 失敗系(timeout/error)を入れる
3) 同時実行制御(Semaphore/Lock)を入れる
4) ログと集計を整える

運用のコツ
- まず動く最小版を作る
- 1機能ずつ追加して毎回実行確認する
- どこで待つか(await位置)を必ず言語化する
- queue.join と gather の役割を分けて考える
